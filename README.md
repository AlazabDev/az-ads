## 🔍 نظرة عامة على المشروع

## ✨ المميزات الرئيسية

### 🎨 1. استوديو توليد الأصول (Asset Generation Studio)
* **توليد وتخصيص الصور والفيديو:** واجهة مستخدم مرنة ومبتكرة مخصصة لإنشاء الأصول الإعلانية، مع خيارات لتحديد المقاسات (مربع، طولي للستوري، عرضي للبانرات).
* **مكتبة الأصول الرقمية (Asset Library):** أرشيف سحابي منظم يحتوي على كل ما قام المستخدم بتوليده، مع إمكانية الفرز، والتصفية، والتحميل المباشر.

### 📊 2. نظام إدارة الحملات (Campaign Management System)
* **منشئ الحملات (Campaign Builder):** نموذج متكامل لتحديد تفاصيل الحملة (اسم الحملة، الميزانية الإجمالية، الميزانية اليومية، الجمهور المستهدف، المنصات المستهدفة).

* **لوحة التحكم والإحصائيات (Dashboard):** شاشة رئيسية تقدم رسوماً بيانية وتلخيصاً دقيقاً للميزانيات المستهلكة، عدد الحملات النشطة، وتوزيع الأصول.

### 🔒 3. الأمن والبنية التحتية (Security & Infrastructure)
* **المصادقة وحماية الهوية (Supabase Auth):** تأمين حسابات المستخدمين عبر البريد الإلكتروني وكلمة المرور، أو التسجيل عبر وسائل التواصل الاجتماعي.

* **التحديث الفوري (Real-time DB):** مزامنة لحظية للتغيرات في حالة الحملات (نشطة، متوقفة مؤقتاً، مسودة) بين قاعدة البيانات والواجهة الأمامية.

## 📊 تصميم قاعدة البيانات (Supabase Schema)

لضمان عمل الاستوديو وإدارة الحملات بشكل مترابط، نقترح الهيكل التالي للجداول في قاعدة بيانات PostgreSQL عبر Supabase:

### 1. جدول ملفات المستخدمين (`profiles`)

* `id` (uuid, Primary Key) -> يربط بـ `auth.users` تلقائياً.
* `updated_at` (timestamp)
* `full_name` (text)
* `company_name` (text)

### 2. جدول الأصول المولدة (`assets`)

* `id` (bigint, Primary Key)
* `created_at` (timestamp)
* `user_id` (uuid, Foreign Key -> profiles.id)
* `asset_type` (text) -> يحدد نوع الأصل (`image` أو `video`).
* `url` (text) -> رابط الملف المخزن في الـ Supabase Storage Bucket.
* `prompt_used` (text) -> النص المستخدم لتوليد هذا الأصل للرجوع إليه لاحقاً.

### 3. جدول الحملات (`campaigns`)

* `id` (bigint, Primary Key)
* `created_at` (timestamp)
* `user_id` (uuid, Foreign Key -> profiles.id)
* `name` (text) -> اسم الحملة الإعلانية.
* `budget` (numeric) -> الميزانية المخصصة.
* `status` (text) -> حالة الحملة (`active`, `paused`, `draft`, `completed`).
* `start_date` (date)
* `end_date` (date)

### 4. جدول الربط بين الحملات والأصول (`campaign_assets`)

* `id` (bigint, Primary Key)
* `campaign_id` (bigint, Foreign Key -> campaigns.id)
* `asset_id` (bigint, Foreign Key -> assets.id)
